# リアリティショー シーン検索システム

**話者ごとの音声**を同じ時間軸で文字起こしし、「誰がどういったことを言っているところか」で検索すると、**誰の音声の何秒で**その発言があるかをトップ3（または指定件数）で返すツールです。

- 例: その場に6人いれば、6本の音声（各話者1本）を用意し、すべて同じ時間軸で同期されている想定。
- 検索結果は「**田中 の 120.5秒〜135.2秒 で 「…」 と言っています**」のように、**誰が・何秒で・何を言ったか**が分かる形式で表示されます。

## 必要な環境

- Python 3.10+
- **FFmpeg**（音声・動画の読み込み用。未導入の場合は `brew install ffmpeg` などでインストール）
- （GPU があれば Whisper が高速になりますが、CPU のみでも動作します）

## セットアップ

```bash
cd reality_edit
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

- 初回の検索・インデックス作成時には、Whisper と sentence-transformers のモデルが自動ダウンロードされます。

## ブラウザから使う（Web）

音声のアップロードと検索をブラウザで行う場合は、Web サーバーを起動します。

```bash
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

ブラウザで **http://localhost:8000** を開くと次のことができます。

1. **音声をアップロード** … 話者ごとの音声ファイルを複数選択し、話者名をカンマ区切りで入力してアップロード
2. **インデックスを作成** … アップロードした音声を文字起こしし、検索用インデックスを構築（数分かかることがあります）
3. **シーンを検索** … クエリを入力して「誰が・何秒で・何を言ったか」を表示

## 使い方（CLI）

### 1. インデックス作成（話者付き音声の文字起こし＋検索用データの構築）

話者ごとの音声ファイルを、**同じ時間軸で録音されたもの**として指定します。  
例: 6人なら6本の音声を `path:話者名` の形式で渡します。

```bash
python main.py index --sources "path/to/田中.wav:田中" "path/to/佐藤.wav:佐藤" "path/to/鈴木.wav:鈴木"
```

- 話者名に `:` を含めたい場合は、`--audios` と `--speakers` を併用してください。

```bash
python main.py index --audios person1.wav person2.wav person3.wav --speakers 田中 佐藤 鈴木
```

- `--model`: Whisper のモデルサイズ（`base` / `small` / `medium` / `large`）。デフォルトは `small`。
- `--language`: 言語コード。日本語なら `ja`（デフォルト）。

結果は `data/transcripts/`（話者ごとの文字起こし JSON）と `data/index/`（ChromaDB）に保存されます。

### 2. シーン検索（誰が・何秒で・何を言ったか）

自然言語でクエリを入力すると、意味的に近いシーンを**話者とタイムスタンプ付き**で表示します。

```bash
python main.py search "誰がどういったことを言っているところか"
```

- `--top N`: 表示する件数（デフォルト 3）。

出力例:

| 順位 | 話者 | 開始(秒) | 終了(秒) | スコア | 発言内容（抜粋） |
|------|------|----------|----------|--------|------------------|
| 1    | 田中 | 120.5    | 135.2    | 85.00% | その時に彼が…    |
| 2    | 佐藤 | 89.0     | 102.1    | 78.00% | あの人がこう…   |

その下に、「**田中 の 120.5秒〜135.2秒 で 「…」 と言っています**」のように、**誰の音声の何秒のところで言っているか**がひと目で分かる形で表示されます。

## 仕組み

1. **文字起こし**: 話者ごとの音声を **faster-whisper** で認識し、セグメントごとの開始・終了秒数を取得。全ファイルが同じ時間軸で同期されているため、秒数は共通タイムラインとして扱える。
2. **シーン分割**: 短い区間をまとめて「シーン」単位にし、**誰が・何秒で・何を言ったか**を紐づけて保存。
3. **意味検索**: シーンのテキストを **sentence-transformers**（多言語対応）でベクトル化し、**ChromaDB** に保存。検索時はクエリも同じモデルでベクトル化し、類似度の高いシーンをトップ K 件返す。各ヒットには話者と開始・終了秒が付く。

## 設定

`config.py` で以下を変更できます。

- `WHISPER_MODEL`: 文字起こしのモデルサイズ
- `EMBEDDING_MODEL`: 意味検索用の埋め込みモデル（日本語対応のものを推奨）
- `TOP_K`: 検索時のデフォルト件数（3）
- `DATA_DIR` / `INDEX_DIR` / `TRANSCRIPTS_DIR`: データの保存先

## ディレクトリ構成

```
reality_edit/
├── app.py            # Web サーバー（uvicorn app:app）
├── main.py           # CLI エントリポイント
├── config.py         # 設定
├── requirements.txt
├── static/
│   └── index.html   # ブラウザ用 UI（アップロード・検索）
├── src/
│   ├── transcribe.py   # 音声の文字起こし（Whisper）
│   ├── indexer.py      # シーン分割・話者付きインデックス構築
│   └── search_engine.py # 「誰が何秒で何を言ったか」で検索
└── data/             # 実行後に生成
    ├── uploads/      # アップロードした音声と mapping.json
    ├── transcripts/  # 話者ごとの文字起こし JSON
    └── index/        # ChromaDB と scenes.json
```

## 注意

- 各話者の音声は**同じ長さ・同じ時間軸**で用意してください（例: 全員分を同じ収録の別トラックとして出力）。
- 音声が長いと文字起こしに時間がかかります。最初は短いクリップで試すとよいです。
